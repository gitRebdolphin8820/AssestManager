## 一、项目概述

本项目是一个**个人资产管理系统**，用于管理个人财务资产、负债、基金投资、黄金资产等。系统采用前后端分离架构，支持数据持久化存储。

### 1.1 项目结构

项目包含三个主要目录：

```
fundWeb/
├── asset-managerVue/          # Vue 前端项目（当前使用版本）
├── backendVue/                # Node.js 后端项目
└── asset-manager/             # 旧版纯前端项目（已废弃）
```

> **注意**：`asset-manager` 是旧版本，使用浏览器本地存储，现已废弃。当前使用 `asset-managerVue` + `backendVue` 版本。

## 五、数据库设计

### 5.1 数据表总览

| 表名 | 用途 | 记录数预估 | 主要关联 |
|------|------|-----------|----------|
| `accounts` | 账户信息（银行卡、支付宝等） | 5-20 条 | 主表 |
| `assets` | 资产信息（存款、基金、股票等） | 10-50 条 | 关联 accounts |
| `gold_assets` | 黄金资产详情 | 0-10 条 | 关联 accounts |
| `funds` | 基金投资详情 | 0-30 条 | 独立表 |
| `debts` | 负债信息（房贷、车贷等） | 0-10 条 | 独立表 |
| `transactions` | 交易记录（收入、支出、转账） | 100-10000 条 | 关联 accounts/assets |
| `settings` | 系统设置（金价、默认账户等） | 1 条 | 单例表 |

### 5.2 表关系图

```
┌─────────────────────────────────────────────────────────────┐
│                         数据关系图                           │
└─────────────────────────────────────────────────────────────┘

accounts (账户表) 【主表】
    │
    ├── 1:N ── assets (资产表)
    │              │
    │              ├── 子类型: current (活期存款)
    │              ├── 子类型: fixed (定期存款)
    │              ├── 子类型: fund (基金)
    │              ├── 子类型: stock (股票)
    │              └── ...
    │
    ├── 1:N ── gold_assets (黄金资产表) 【独立表】
    │
    └── 1:N ── transactions (交易记录表)
                  │
                  ├── 类型: income (收入)
                  ├── 类型: expense (支出)
                  └── 类型: transfer (转账)

funds (基金表) 【独立表】
    └── 记录每只基金的持仓详情

debts (负债表) 【独立表】
    └── 记录房贷、车贷等负债

settings (设置表) 【单例表，仅1条记录】
    └── 存储系统级配置
```

#### 5.3.4 funds（基金表）

存储基金投资详情，独立表不关联 assets。

**表结构**：

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | VARCHAR(50) | ✅ | 主键 |
| name | VARCHAR(100) | ✅ | 基金名称 |
| code | VARCHAR(20) | ✅ | 基金代码，如"005827" |
| type | VARCHAR(50) | ❌ | 基金类型（见下方枚举） |
| is_fixed | BOOLEAN | ❌ | 是否固定收益，默认 false |
| cost_nav | DECIMAL(10,4) | ❌ | 成本净值，默认 0 |
| shares | DECIMAL(10,4) | ❌ | 持有份额，默认 0 |
| cost_amount | DECIMAL(15,2) | ❌ | 成本金额，默认 0 |
| current_nav | DECIMAL(10,4) | ❌ | 当前净值，默认 0 |
| sell_nav | DECIMAL(10,4) | ❌ | 卖出净值，默认 0 |
| sell_shares | DECIMAL(10,4) | ❌ | 卖出份额，默认 0 |
| update_date | DATE | ❌ | 更新日期 |
| remark | TEXT | ❌ | 备注 |
| created_at | TIMESTAMP | ✅ | 创建时间 |
| updated_at | TIMESTAMP | ✅ | 更新时间 |

**基金类型枚举**：
- `混合型` - 混合型基金
- `股票型` - 股票型基金
- `债券型` - 债券型基金
- `指数型` - 指数基金
- `QDII` - QDII基金（海外投资）
- `货币型` - 货币基金
- `FOF` - 基金中的基金

**完整数据示例**：

```json
{
  "id": "id_1776132800000_jkl22222",
  "name": "易方达蓝筹精选",
  "code": "005827",
  "type": "混合型",
  "isFixed": false,
  "costNav": 2.5000,
  "shares": 10000.0000,
  "costAmount": 25000.00,
  "currentNav": 3.2000,
  "sellNav": 0,
  "sellShares": 0,
  "updateDate": "2025-04-14",
  "remark": "张坤管理，长期持有",
  "createdAt": "2025-04-14T12:30:00.000Z",
  "updatedAt": "2025-04-14T18:30:00.000Z"
}
```

**收益计算**：
- 当前市值 = 当前净值 × 持有份额
- 累计收益 = 当前市值 - 成本金额
- 收益率 = (累计收益 / 成本金额) × 100%

## 八、常见问题排查

### 8.1 后端启动失败

**问题**：`Error: connect ECONNREFUSED 127.0.0.1:5432`

**解决**：
1. 检查 PostgreSQL 服务是否启动
2. 检查数据库配置是否正确（用户名、密码、数据库名）
3. 确认数据库 `asset_manager` 已创建

### 8.2 前端无法连接后端

**问题**：`API请求失败: 500` 或网络错误

**解决**：
1. 确认后端服务已启动（`node server.js`）
2. 检查 `vite.config.js` 中的代理配置
3. 检查浏览器控制台网络请求

### 8.3 数据库表缺失

**问题**：`relation "xxx" does not exist`

**解决**：
1. 重启后端服务，会自动创建表
2. 或手动执行 `server.js` 中的建表 SQL

### 8.4 端口被占用

**问题**：`Port 3000 is already in use`

**解决**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :3000

# 结束进程（Windows）
taskkill /PID <进程ID> /F
```
