# 统一资产管理系统 - 使用说明

## 项目概述

这是一个集大成的资产管理系统，可以把不同账户的不同资产，不同账户的各种流水，全让它们联动起来，不再需要打开一个又一个软件，记账本永远对得上。

## 系统特点

### 1. 数据联动
- **账户-资产联动**：所有资产都关联到具体账户，自动计算账户余额
- **交易-资产联动**：每次交易自动更新相关资产价值
- **流水-交易联动**：流水记录作为交易记录的子集，确保数据一致性

### 2. 功能模块
- **资产管理**：按类型或按账户查看所有资产
- **基金管理**：支持导入导出、净值更新、收益计算、分页显示
- **黄金管理**：按克数记录，自动计算价值
- **负债管理**：记录房贷、车贷、信用卡等负债
- **交易管理**：统一管理所有交易类型，包含流水记录功能，支持卡片视图和表格视图，表格视图支持分页显示


### 3. 数据模型
- **账户数据**：银行卡、微信、支付宝等账户信息
- **资产数据**：活期、定期、基金、黄金、股票、债券、保险等
- **交易数据**：收入、支出、投资、定期等所有交易类型
- **流水数据**：作为交易记录的子集，专注于日常收支

## 如何使用

### 1. 启动系统

#### 方法一：使用Python HTTP服务器（推荐）
```bash
cd d:\fundWeb\asset-manager
python -m http.server 8000
```

#### 方法二：使用Node.js HTTP服务器
```bash
cd d:\fundWeb\asset-manager
npx http-server -p 8000
```

#### 方法三：直接打开HTML文件
直接在浏览器中打开 `d:\fundWeb\asset-manager\index.html` 文件

### 2. 访问系统
在浏览器中访问：`http://localhost:8000`

### 3. 使用流程

#### 第一步：创建账户
1. 点击"资产"标签页
2. 点击"新增资产"按钮
3. 在弹窗中选择"创建新账户"
4. 填写账户信息（如：招商银行、支付宝、微信等）
5. 保存账户

#### 第二步：添加资产
1. 在"资产"标签页中，点击"新增资产"
2. 选择已创建的账户
3. 选择资产类型（活钱、定期、基金、黄金等）
4. 填写资产信息
5. 保存资产

#### 第三步：记录流水
1. 点击"交易"标签页
2. 点击"记一笔"按钮
3. 选择收入或支出类型
4. 填写金额、分类、账户、资产等信息
5. 保存记录
6. 系统会自动更新相关账户和资产余额

#### 第四步：管理基金
1. 点击"基金"标签页
2. 可以手动添加基金或导入Excel数据
3. 更新基金净值查看收益情况
4. 导出数据进行备份

#### 第五步：查看统计
1. 在"资产"标签页查看资产分布图表
2. 在"交易"标签页查看收支统计和所有交易记录

## 数据存储

所有数据都存储在浏览器的localStorage中，数据包括：
- 账户信息（accounts）
- 资产信息（assets）
- 黄金资产（goldAssets）
- 基金数据（fundData）
- 负债数据（debts）
- 交易记录（transactions）
- 系统设置（settings）

## 技术架构

### 前端技术
- HTML5：页面结构
- CSS3：样式设计
- JavaScript：业务逻辑
- Chart.js：数据可视化

### 模块化设计
- `dataManager.js`：统一数据管理模块
- `asset.js`：资产管理模块
- `fund.js`：基金管理模块
- `gold.js`：黄金管理模块
- `debt.js`：负债管理模块
- `flow.js`：流水管理模块（已整合到交易管理中）
- `transaction.js`：交易管理模块（包含流水管理功能）
- `utils.js`：工具函数模块
- `script.js`：主脚本文件

### 数据联动机制
所有数据变动都通过交易记录驱动，确保：
- 数据可追溯性：每笔资产变动都有对应的交易记录
- 数据一致性：所有资产余额都基于交易记录计算
- 数据完整性：不会出现数据不一致的情况

## 常见问题

### Q: 为什么我的流水记录没有更新资产余额？
A: 请确保在记录流水时选择了正确的账户和资产，系统会自动更新相关资产余额。

### Q: 如何备份数据？
A: 可以通过以下方式备份数据：
1. 导出基金数据（在基金页面）
2. 复制localStorage数据（在浏览器开发者工具中）
3. 定期截图保存重要数据

### Q: 如何恢复数据？
A: 可以通过以下方式恢复数据：
1. 导入基金数据（在基金页面）
2. 粘贴localStorage数据（在浏览器开发者工具中）

### Q: 系统支持多用户吗？
A: 目前系统是单用户设计，数据存储在本地浏览器中。如需多用户支持，需要后端服务器。

### Q: 如何清除所有数据？
A: 在浏览器开发者工具的Application标签中，找到localStorage，清除所有数据。
